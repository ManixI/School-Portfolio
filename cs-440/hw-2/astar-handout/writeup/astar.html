AStar

In this assignment, you'll implement the A* Search algorithm. The stub code provides the framework for running A* on a generic `SearchProblem`. Two `SearchProblem` classes are (partially) given: both work on simple 2D grids: the GridProblem class works on a grid represented by a list of lists (an example is included); the ImageProblem class works on a grid represented as pixels from an image.

Notes on the Code

I've added some descriptive doc-strings and comments. My hope is that these will help you understand the code better and write more idiomatic python. Here are a few items worth calling out: heapq the heapq module implements heap operations on a list. It is worth reading about. It let's you easily push and pop on a heap as well as heapify an unsorted list (recall that a fully sorted list would also be in heap order). namedtuple named tuples create objects that are immutable (like tuples) but whose data can be accessed by name (instead of by an integer index as in a normal tuple). The TNode class should be used to describe a node in the search tree and thus a path from the one state to another. The syntax for creating a namedtuple class is relatively strange so there is a comment to explain it. However, aside from declaring the TNode class itself, usage is straightforward. To create a new TNode instance, you simply call the class supplying arguments for each instance variable. 
  a = TNode( 10, 6, 4, (0,0), None, "L" )
As mentioned above, you can (and should) access data from TNode using names of the instance variables; so the following statements: 
  print(a.f)
  print(a.g)
  print(a.state)
  print(a.parent)
  print(a.action)
would print: 
  10
  6
  (0, 0)
  None
  L
